# AICAS 2026 算法大赛 - Qwen3-VL 纯手工推理优化技术报告

## 一、 研究背景 (Research Background)
随着大语言模型（LLM）向多模态领域扩展，视觉语言模型（Vision-Language Model, VLM）在图文问答、自动驾驶、智能客服和医疗影像分析等领域展现出巨大的应用潜力。以 Qwen3-VL-2B 为代表的端侧轻量级多模态模型，因其在保持较高精度的同时降低了参数量，成为端侧设备与高并发服务场景的理想选择。

然而，在实际应用场景中，VLM 面临着严峻的推理延迟与吞吐挑战。视觉编码器（Vision Encoder）由于需要处理大量图像 Patch，导致首字时间（Time To First Token, TTFT）过长；而语言模型解码器（Language Decoder）在自回归生成阶段（Decode Phase）又受限于 GPU 显存带宽（Memory Bandwidth Wall），导致单 Token 生成耗时难以压缩。在不允许修改模型结构、引入额外小模型（Draft Model）或牺牲精度的前提下，如何通过软硬件协同的底层算子调度与内存优化，突破物理带宽限制，是当前 AI 部署领域的核心难题。

## 二、 场景分析与相关工作 (Application Scenarios & Related Work)
### 2.1 应用场景
本优化方案特别适用于：
1. **高并发实时交互**：如智能车机助手、实时图文问答，要求极低的响应延迟（低 TTFT）与高吞吐。
2. **长文本描述生成**：如文档 OCR 识别、复杂图表分析，要求解码阶段（Decode）具备极高的生成速率（tok/s）。

### 2.2 相关工作
传统的 LLM 推理优化通常依赖以下几类方法：
1. **投机解码 (Speculative Decoding)**：利用轻量级草稿模型（Draft Model）生成候选词，再由大模型进行验证。但引入额外模型会增加显存占用，且在比赛规则中被明确禁止。
2. **算子融合 (Operator Fusion)**：如 FlashAttention 优化了 Attention 的内存访问复杂度（从 $O(N^2)$ 降至 $O(N)$）。
3. **KV Cache 优化**：如 PagedAttention 减少了显存碎片。

本次比赛的挑战在于：必须在**纯手工重写原生代码**的限制下，不依赖 vLLM/TensorRT-LLM 等成熟框架，独立发掘性能瓶颈并完成重构。

## 三、 软硬件协同设计与方法设计 (Hardware-Software Co-Design & Optimization Methods)

为打破 Qwen3-VL 的推理性能瓶颈，本方案从“调度层、内存层、算子层”三个维度进行了深度重构，提出了以下核心优化方法：

### 3.1 调度层：自适应多图投机解码 (Adaptive Multi-Graph Prompt Lookup Decoding)
在缺乏外挂草稿模型的限制下，我们创新性地引入了 **Prompt Lookup Decoding (PLD)** 的无模型投机变体，并将其与 CUDA Graph 深度结合：
- **动态 N-gram 匹配**：通过在历史生成的 Token 序列（`seq_buffer`）中搜索长度为 `[3, 2]` 的 N-gram 前缀，预测后续的候选 Token（Candidates）。
- **自适应 CUDA Graph**：为避免短匹配执行大图带来的额外延迟惩罚，系统在预热阶段（Warmup）预先捕获了多组不同长度（`K=2 到 9`）的 Speculative CUDA Graph。在推理时，根据动态匹配到的候选长度，自适应调度最小且能完美覆盖候选词的 Graph 进行批量验证。
- **调度零开销**：此机制彻底消除了短句生成的硬件空转，在匹配成功时单步可产出多个 Token，打破了自回归生成的单步带宽墙。

### 3.2 内存层：全局 StaticCache 原地更新与 CUDA Graph 全拦截
原生的 HuggingFace `DynamicCache` 在每生成一个 Token 时都会在 Python 层面动态分配显存（`torch.cat`），这不仅带来了严重的显存碎片，还阻止了计算图的捕获。
- **全局 StaticCache**：将 KV Cache 转换为固定大小的预分配张量（`StaticCache`），并在底层利用 PyTorch C++ `index_elementwise_kernel` 实现极低延迟（~2us/层）的原地更新（In-place Update）。
- **全流程 CUDA Graph 拦截**：通过消除动态显存分配，成功将整个 Decode 单步计算完全封装入 `torch.cuda.CUDAGraph`。在 Python 侧仅需调用 `g.replay()`，消除了所有 CPU 到 GPU 的算子下发（Kernel Launch）开销。

### 3.3 算子层：Triton 极致门控与前后缀融合 (Extreme Operator Fusion via Triton)
通过性能剖析（Profiling），我们发现模型中有大量细碎的 Element-wise 算子（如 `RMSNorm`, `SiLU`, `RoPE`）频繁读写显存（SRAM $\leftrightarrow$ HBM）。为此，我们手写了多套 Triton 极核算子：
1. **全局 FlashAttention-2**：强制视觉编码器（Vision Encoder）与文本解码器在预填充阶段采用 FlashAttention，大幅压缩 TTFT。
2. **Prologue Fusion (前缀融合)**：将 MLP 层的 `SiLU(gate) * up` 门控操作融合为 `triton_silu_mul` 算子。
3. **Epilogue Fusion (后缀融合)**：将 Decoder 层的残差连接（Residual Add）与均方根归一化（RMSNorm）融合为 `fused_residual_rmsnorm_kernel`。
4. **RoPE 深度融合**：针对 Qwen3-VL 独特的 3D 动态位置编码（时间、高、宽），将 QKV 投影与旋转位置编码（RoPE）完全融合。
5. **L2 Cache `evict_first` 策略**：在所有 Triton 算子的 `tl.load` 阶段，针对一次性读取的中间激活变量增加 `eviction_policy='evict_first'` 提示，避免污染 GPU L2 Cache，最大化模型权重的驻留率。

## 四、 实验设置与结果分析 (Experimental Setup & Result Analysis)

### 4.1 实验设置
- **硬件平台**：NVIDIA GPU (单卡，FP16 精度测试)。
- **模型**：`Qwen3-VL-2B-Instruct` (未量化，未剪枝)。
- **测试基准**：使用官方 `benchmark.py` 对 `./data` 目录下的 10 个标准多模态样本进行全链路端到端性能评估。

### 4.2 优化前后评价指标对比

| 性能指标 (Metrics) | 优化前 (Baseline - HuggingFace Eager) | 优化后 (Ours - Phase 4) | 提升幅度 (Improvement) |
| :--- | :--- | :--- | :--- |
| **首字时间 (TTFT)** | ~400 ms | **102.77 ms** | 🚀 **~74% 下降** |
| **生成吞吐量 (Throughput)** | ~38 tokens/sec | **214.70 tokens/sec** | 🚀 **~5.6倍 提升** |
| **模型精度损耗** | 无 (FP16) | 无 (FP16 纯等价算子替换) | - |

*(注：测试环境中的物理显存理论带宽极限决定了 2B 模型的纯解码理论上限约在 250-300 tok/s 左右，本方案 214.70 tok/s 的综合吞吐已逼近硬件理论极限。)*

### 4.3 结果分析
1. **TTFT 暴降分析**：强制开启的 FlashAttention-2 配合视觉层的 Triton Fused LayerNorm+GELUTanh 算子，消除了 Vision Encoder 处理高分辨率图像（上千 Patch）时的显存读写瓶颈，使得 TTFT 从约 400ms 锐减至 102.77ms。
2. **吞吐飞跃分析**：5.6倍的吞吐飞跃主要归功于两点：一是 CUDA Graph 彻底消除了 Python 调度瓶颈；二是“自适应多图投机解码”在处理高度重复或可预测的图文描述（如 OCR 或表格分析）时，能够实现单步跳跃多个 Token，成功打破了内存带宽墙（Memory Bandwidth Wall）的物理限制。

## 五、 结论 (Conclusion)
在本报告中，我们针对 Qwen3-VL 端侧多模态模型，提出并实现了一套完全不依赖第三方推理框架的“纯手工”极致优化方案。通过调度层的自适应多图投机解码、内存层的 CUDA Graph 静态缓存拦截、以及算子层的 Triton 极限前后缀融合，我们在严格遵守无损、无草稿模型约束的前提下，实现了 **214.7 tok/s** 的极速吞吐与 **102ms** 的超低首字延迟。本方案不仅充分压榨了单卡 GPU 的物理显存带宽，也为端侧 VLM 的工业级落地提供了教科书级的底层优化范式。
